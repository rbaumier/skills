# Data-Test Attribute Strategy

## When `data-test` Attributes Beat `getByRole()`

- Custom component libraries with non-standard ARIA mappings
- Third-party components where role selectors are unreliable or unstable across versions
- Complex composite widgets without clear ARIA roles
- Dynamic elements generated by frameworks (e.g., AG Grid cells, chart elements)

When standard HTML semantics are correct and accessible, prefer `getByRole()`. Fall back to `data-test` when role-based selectors would be brittle or ambiguous.

## Configuration

Configure `testIdAttribute` in `playwright.config.ts` to match your project's convention:

```typescript
// playwright.config.ts
export default defineConfig({
  use: {
    testIdAttribute: 'data-test',
  },
});
```

## Usage

Use `page.getByTestId()` instead of raw CSS selectors:

```typescript
// GOOD — uses configured testIdAttribute
await page.getByTestId('login-form');

// BAD — raw CSS selector, bypasses config
await page.locator('[data-test="login-form"]');
```

## Naming Conventions

- **kebab-case**: `login-email-input`, not `loginEmailInput`
- **Feature-prefixed**: group related elements by feature
  - `login-email-input`
  - `login-password-input`
  - `login-submit-button`
  - `subscription-url-input`
  - `subscription-events-table`

## When to Fall Back to Role-Based Selectors

- Standard HTML buttons, links, headings, form inputs with proper labels
- Navigation landmarks (`<nav>`, `<main>`, `<header>`)
- Elements with explicit ARIA roles that are stable and well-defined

## Example: Before/After

```typescript
// BEFORE — brittle CSS selectors
await page.locator('.login-form input[type="email"]').fill('user@test.com');
await page.locator('.login-form input[type="password"]').fill('pass');
await page.locator('.login-form button.submit').click();

// AFTER — data-test attributes with getByTestId
await page.getByTestId('login-email-input').fill('user@test.com');
await page.getByTestId('login-password-input').fill('pass');
await page.getByTestId('login-submit-button').click();
```
