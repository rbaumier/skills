## Highest-Value Rules (FAIL without skill, PASS with skill)

These 28 rules are where the skill provides the most value -- they would not be followed based on general knowledge alone:

### class-management (5 rules)
- **use-cn**: Use cn() helper (twMerge + clsx) -- general knowledge only suggests clsx
- **use-cva**: Use cva() for variants -- general knowledge suggests @apply
- **avoid-apply**: Avoid @apply for components -- general knowledge recommends @apply
- **class-order**: Specific ordering convention -- general knowledge gives vague advice
- **tailwind-merge**: Explicit conflict resolution tool -- not mentioned without skill

### responsive-mobile (1 rule)
- **container-queries**: @container for component-level responsiveness -- not mentioned without skill

### spacing-sizing (2 rules)
- **spacing-scale**: Multiples of 4 only -- general knowledge accepts p-5, mb-7
- **size-shorthand**: size-* instead of w-* h-* -- not mentioned without skill

### semantic-tokens (4 rules)
- **oklch-format**: OKLCH color format -- general knowledge uses hex
- **color-mix**: color-mix() in oklab -- not known without skill
- **clear-defaults**: --color-*: initial -- not known without skill
- **theme-block**: @theme block for tokens -- general knowledge uses config file

### accessibility (2 rules)
- **touch-targets**: 44x44px minimum -- not mentioned without skill
- **contrast-more**: contrast-more: variant -- not known without skill

### animations-perf (4 rules)
- **gpu-only**: GPU-only properties, no transition-all with w/h -- weakly addressed without skill
- **skeleton-pattern**: Full skeleton pattern with dark mode -- incomplete without skill
- **starting-style**: @starting-style for popover animations -- not known without skill
- **animation-tokens**: --animate-* in @theme -- not known without skill

### dark-mode (2 rules)
- **v4-dark-builtin**: @custom-variant dark (&:where(.dark, .dark *)) -- vague without skill
- **semantic-body**: bg-background text-foreground on body, border-border on * -- not mentioned without skill

### v4-migration (5 rules)
- **v4-native-animations**: Replace tailwindcss-animate with native CSS -- not known without skill
- **v4-utility-directive**: @utility directive -- not known without skill
- **v4-custom-variant**: @custom-variant directive -- not known without skill
- **v4-build-integration**: @tailwindcss/postcss or @tailwindcss/vite -- vague without skill
- **content-scope**: Never include node_modules in content -- not flagged without skill

### layout-patterns (1 rule)
- **grid-overflow**: min-w-0 + truncate in grid cells -- min-w-0 missed without skill

### components-react (4 rules)
- **compound-component**: Compound component pattern -- not suggested without skill
- **radix-primitives**: Radix for accessible primitives -- not mentioned without skill
- **react-19-ref**: No forwardRef in React 19 -- not flagged without skill

## Rules That Pass Without Skill (General Knowledge)

These 16 rules are well-known enough to be followed without the skill:

- mobile-first design (responsive-mobile)
- responsive text scaling (responsive-mobile)
- prefer text-base over text-[16px] (spacing-sizing)
- semantic tokens over raw colors (semantic-tokens)
- CSS variable-based theming (semantic-tokens)
- focus styles on interactive elements (accessibility)
- sr-only / aria-label for icon buttons (accessibility)
- motion-reduce for animations (accessibility)

- systematic z-index (layout-patterns)
