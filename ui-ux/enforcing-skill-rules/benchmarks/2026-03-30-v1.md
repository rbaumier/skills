## Discriminating Assertions (36 — only pass with skill)

These are the rules where the skill provides unique, measurable value. Grouped by theme:

### Anti-AI-Slop (6 assertions, 100% discrimination)
- `no-3-col-equal-cards` — LLMs default to `repeat(3, 1fr)` every time
- `no-ai-color-palette` — cyan/purple on dark is the #1 AI tell
- `no-gradient-text-or-accent` — default decorative gradient instinct
- `no-hero-metric-layout` — big number + small label is a universal template
- `no-centered-text-over-dark` — centered hero is the default LLM layout
- `no-glassmorphism` — backdrop-filter blur is reflex for "modern" UI

### Color System (6 assertions, 100% discrimination)
- `oklch-color-space` — LLMs default to hex; OKLCH never appears without guidance
- `tinted-neutrals` — untinted grey scale is universal default
- `no-pure-black-white` — #000 and #fff appear in every baseline output
- `no-pure-black-bg` — same as above for backgrounds
- `60-30-10-rule` — mentioned vaguely without skill, structured with it
- `dark-mode-strategy` — swapping to #000/#fff is the default, not lighter-surfaces-for-depth

### Shadow System (6 assertions, 100% discrimination)
- `layered-shadows` — single shadow per level is universal default
- `no-pure-black-shadows` — rgba(0,0,0,...) is always the default
- `shadow-tokens-scale` — Tailwind naming (sm/md/lg) instead of semantic --shadow-1/2/3
- `animate-via-pseudo` — `transition: box-shadow` is the instinctive approach
- `consistent-light-source` — never considered without skill
- `concentric-radius` — never mentioned without skill

### Typography & Visual Weight (4 assertions, 100% discrimination)
- `distinctive-font-choice` — Inter/Roboto is always the default pick
- `dramatic-size-contrast` — weak 1.5-2x jumps without guidance
- `weight-contrast` — 600/400 default, not 800/300
- `semantic-token-naming` — flat naming (--primary-50) without skill

### Form Quality (5 assertions, 83% discrimination)
- `correct-input-types` — type="text" for email is common mistake
- `button-for-actions` — div onClick appears without skill
- `verb-noun-buttons` — "Submit" is universal default
- `inline-errors-focus` — alert() is the fallback pattern
- `error-messages-include-fix` — terse "required" messages without guidance
- `touch-targets-44px` — 8px padding producing ~32px targets

### Empty State & Copy Quality (5 assertions, 80% discrimination)
- `what-will-be-here` — "No tasks yet" (negation) is the default
- `why-it-matters` — value proposition absent or slop-filled
- `visual-interest` — emoji icons instead of custom illustration
- `no-ai-slop-copy` — "unleash", "seamless", "elevate" appear freely
- `active-voice-copy` — passive constructions dominate

### Hero & Layout (3 assertions, 100% discrimination)
- `no-bounce-easing` — bounce animation is default "polish"
- `no-h-screen` — 100vh instead of 100dvh
- `action-hierarchy` — gradient CTAs without hierarchy reasoning
