## Key learnings

1. Flat bullet lists lose rules — sectioning + bolding discriminating rules works
2. "In reviews: if you see X, flag it" checklists are the most effective pattern
3. Concrete code snippets (before→after) beat abstract descriptions
4. Targeted prompts (1 assertion per prompt) isolate and fix flaky rules faster
5. Non-discriminating rules (both pass/fail equally) can be compressed safely
