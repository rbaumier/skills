### ✅ Migration Strategy (Expand-and-Contract for Breaking Changes)

```sql
-- If renaming columns in production, never use ALTER TABLE ... RENAME (breaks running instances).
-- Use 5-step expand-and-contract pattern:

-- Step 1: Add new nullable column (separate migration)
ALTER TABLE public."user" ADD COLUMN full_name_new TEXT;

-- Step 2: Backfill in batches (separate, can be interrupted safely)
UPDATE public."user" SET full_name_new = full_name WHERE full_name_new IS NULL LIMIT 1000;
-- Repeat until no rows updated (idempotent: add WHERE clause to skip already-migrated)

-- Step 3: Dual-write in app (write both name + full_name_new)
-- Step 4: Switch reads to full_name_new
-- Step 5: Drop old column (separate migration, weeks later, after verification all instances switched)
```

```typescript
// ✅ OTel logger: attach to Drizzle client for query observability
// Zero overhead when no active span (check span existence first)
class OtelDrizzleLogger {
  logQuery(query: string, params: unknown[]): void {
    const span = trace.getActiveSpan();
    if (!span) return;  // Zero overhead when not in trace context
    
    span.setAttribute('db.system', 'postgresql');
    span.setAttribute('db.statement', query);
    span.setAttribute('db.params.count', params?.length ?? 0);
  }
}

export const db = drizzle(pool, {
  schema,
  logger: new OtelDrizzleLogger(),  // Always attach logger for observability
});

// Graceful shutdown: drain pool on app termination
export async function closeDb(): Promise<void> {
  await pool.end();
}
```

```typescript
  readonly _tag = 'TaskCreateError';
  constructor(readonly message: string, readonly cause?: unknown) {}

  readonly _tag = 'DatabaseError';
  constructor(readonly message: string, readonly cause?: unknown) {}

export type TaskRepositoryError = TaskNotFoundError | TaskCreateError | DatabaseError;

  /**

  async findById(id: string): Promise<typeof task.$inferSelect | TaskRepositoryError> {
    try {
      const tasks = await db
        .select({
          // Explicit columns (not SELECT *): enables covering indexes
          id: task.id,
          projectId: task.projectId,
          assigneeId: task.assigneeId,
```

## Issue Summary

### 🔴 Critical Issues Fixed

1. **TRUNCATE** → Use `DELETE FROM` during traffic (TRUNCATE takes ACCESS EXCLUSIVE lock, not MVCC-safe)

2. **PKs: gen_random_uuid()** (UUIDv4 banned) → **UUIDv7** in app layer via `uuidv7()` package

3. **PKs: SERIAL** (32-bit, poor locality) → **UUIDv7** (time-sortable, 128-bit)

4. **varchar(n) / char(n)** → **TEXT** (or CITEXT for case-insensitive)

5. **TIMESTAMP (no tz)** → **TIMESTAMPTZ** (timezone-aware, UTC)

6. **FLOAT for money** → **NUMERIC(12, 2)** (arbitrary precision, no loss)

7. **SELECT \*** → **Explicit columns** (enables covering indexes)

8. **Soft delete: is_deleted boolean** → **is_active boolean** (prefer true by default) or **deleted_at TIMESTAMPTZ** with partial unique index

9. **FK columns not indexed** → **Index all FK columns** (prevents cascade lock contention)

10. **RENAME COLUMN in production** → **Expand-and-Contract**: 5-step dual-write pattern (breaking change)

11. **LIKE '%term%'** on large tables → **ILIKE with parameterization** (or TSVECTOR + GIN for >10K rows)

12. **Offset-based pagination** → **Cursor-based (keyset)** pagination (O(1) vs O(N))

13. **ON DELETE/UPDATE not specified** → **Explicit ON DELETE CASCADE / SET NULL / RESTRICT**

14. **No CHECK constraints** → **CHECK** to make invalid data unrepresentable

15. **No RLS** + superuser connection → **RLS policies enabled** + **app_user role** (least privilege)

16. **drizzle-kit output as-is** → **Manual review**: UUIDv7, TIMESTAMPTZ, CHECK, indexes in third argument

17. **SELECT-then-INSERT** → **INSERT ... ON CONFLICT (UPSERT)** (idempotent, race-condition-safe)

18. **String concatenation in SQL** → **Parameterized queries** (prevents SQL injection)

19. **Throwing errors** → **Errors as values**: typed error union (exhaustive pattern matching)

20. **N+1 query pattern** (loop + nested query) → **Single JOIN query** (eager loading)

21. **autovacuum disabled** → **Never disable** (tuning frequency is fine)

22. **Pool exhaustion risk** (5 × 20 = 100 connections) → **PgBouncer in transaction mode** (not raising max_connections)

23. **No statement_timeout / idle_in_transaction_session_timeout** → **Per-role timeout settings** (prevent runaway queries)

24. **No logger on Drizzle** → **OTel logger attached** (query observability)

25. **Boolean columns not prefixed** → **is_/has_ prefix** (e.g., `is_active`, `has_subscription`)

26. **Nullable columns without justification** → **Comments explaining why NULL is allowed** (or add NOT NULL)

27. **No column comments** → **Comment every column** (rationale, decisions, plain English)

28. **Index naming inconsistency** → **Standardized: {tablename}_{columnname(s)}_{suffix}** (e.g., `idx_task_project_id`)

29. **FK naming inconsistency** → **{from_table}_{from_col}_{to_table}_{to_col}_fk**

30. **Bare timestamp queries** (`NOW()` in WHERE) → **clock_timestamp()** for wall-clock time; **>= AND <** not BETWEEN (off-by-one risk)
