# G4: My Next.js Server Component fetches user data but it seems stale in production

This could be a caching issue. Next.js has built-in caching mechanisms. Try adding `cache: 'no-store'` to your fetch call:

```tsx
const res = await fetch('https://api.example.com/user', {
  cache: 'no-store'
});
```

You might also want to check if you have any CDN caching or if the API itself returns cached responses. Another option is to use `revalidate` to control how often the data refreshes.
