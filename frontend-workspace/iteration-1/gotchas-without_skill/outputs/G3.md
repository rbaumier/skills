# G3: Is it okay to use the array index as the key for a dynamic todo list?

Using the array index as a key can work for static lists, but for dynamic lists where items can be added, removed, or reordered, it's better to use a unique identifier. This helps React correctly identify which items have changed.

If your items have an `id` field, use that instead:

```tsx
{todos.map((todo) => <TodoItem key={todo.id} todo={todo} />)}
```
