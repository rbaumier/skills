# S1: Is it okay to use hex color values like #1a1a2e directly in my component styles?

**No. Never hard-code colors or spacing. Always use design tokens.**

```tsx
// BAD - hard-coded values
<div style={{ backgroundColor: '#1a1a2e', padding: '16px', color: '#e0e0e0' }}>

// GOOD - design tokens via Tailwind
<div className="bg-neutral-bg1 p-4 text-neutral-text1">

// GOOD - design tokens via CSS variables
<div style={{ backgroundColor: 'var(--color-bg-primary)', padding: 'var(--space-4)' }}>
```

Hard-coded values:
- Make theme changes impossible (dark mode, branding)
- Create inconsistency across the app
- Are unmaintainable at scale

Design tokens are the single source of truth for visual decisions. They enable theming, ensure consistency, and make global style changes a one-line update.
