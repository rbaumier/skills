# P5: Should I wrap everything in useMemo and useCallback right away?

**No. Profile before optimizing. No premature optimization.**

```tsx
// BAD - premature optimization
const name = useMemo(() => user.name, [user.name]); // string assignment doesn't need memoization
const handleClick = useCallback(() => setOpen(true), []); // trivial callback

// GOOD - profile first, then optimize targeted bottlenecks
// 1. Open React DevTools Profiler
// 2. Record an interaction
// 3. Identify components with slow renders (>16ms)
// 4. Apply memoization ONLY to those components
```

With React Compiler (React 19), manual memoization is even less necessary — the compiler handles it automatically. Wrapping everything in useMemo/useCallback adds code complexity and can actually hurt performance if the dependencies change frequently (the comparison cost exceeds the computation cost).

**Performance regressions are bugs** — but fix them with data, not guesswork.
