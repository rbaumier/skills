# G4: My Next.js Server Component fetches user data but it seems stale in production

The issue is that **Next.js `fetch()` in Server Components caches by default in production**. This is a common gotcha — your data fetched during the first request gets cached and served to all subsequent requests.

Fix by opting out of caching:

```tsx
// Option 1: No caching
const res = await fetch('https://api.example.com/user', {
  cache: 'no-store'
});

// Option 2: Time-based revalidation
const res = await fetch('https://api.example.com/user', {
  next: { revalidate: 60 } // revalidate every 60 seconds
});
```

You can also set this at the route segment level with `export const dynamic = 'force-dynamic'` or `export const revalidate = 0`.
