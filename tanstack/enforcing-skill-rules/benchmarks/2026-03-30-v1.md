## Analysis

### High-value rules (FAIL without skill, PASS with)
These 23 rules are where the skill provides the most value — they represent knowledge that general LLM training does not reliably produce:

**v5 Renames** (highest impact — 5 rules): `isPending`, `gcTime`, `throwOnError`, `placeholderData: keepPreviousData`, removed query callbacks. These are the most common v4-to-v5 migration errors.

**v5 Behavioral rules** (6 rules): `initialPageParam` required, no `enabled` on `useSuspenseQuery`, no `fetchStatus` conditional with suspense, `retryOnMount`, `signal` cancellation, throw in `queryFn`.

**TanStack Start conventions** (12 rules): Input validation, POST for mutations, HTTP-only cookies, loader data for hydration, `notFound()`/`redirect()`, `beforeLoad`, `.server.ts`/`.functions.ts` naming, env validation, secrets in errors.

### Already-known rules (PASS both)
8 rules passed without the skill — these are well-established patterns: object syntax, array keys, `invalidateQueries`, `createServerFn`, middleware context, Suspense streaming, shared schemas, `VITE_` prefix.

## Conclusion
The skill achieves **100% pass rate** on all 31 assertions in iteration 1. No rule rewording was needed. The skill's primary value is in v5-specific renames/behavioral changes and TanStack Start conventions — areas where general knowledge consistently fails.
