## Key learnings

1. Abstract rules fail — "prefer X over Y" gets ignored. "When reviewing, ask: does this code have X? If not, recommend Y" works
2. Boolean flags: must explicitly say "ternary is NOT a fix" — model defaults to ternary as "good enough"
3. Input validation: "bound every input" is ignored unless you say "reject, never fallback"
4. Architecture reviews: rules phrased as review checklists ("if you see X, flag it") outperform principles
5. DI must be rule #1 in Functions — position matters, it was ignored when buried in the middle
