# R19_2: Do I still need manual useMemo and useCallback with React Compiler?

**Less manual memoization is needed.** The React Compiler handles memoization automatically — it analyzes your code at build time and inserts the necessary memoization.

```tsx
// Before React Compiler - manual memoization everywhere
const sortedItems = useMemo(() => items.sort(byDate), [items]);
const handleClick = useCallback(() => setOpen(true), []);
const MemoChild = memo(ChildComponent);

// With React Compiler - write plain code, compiler optimizes
const sortedItems = items.sort(byDate);  // compiler memoizes if needed
const handleClick = () => setOpen(true); // compiler memoizes if needed
// No manual memo() wrapping needed
```

You can still use useMemo/useCallback for edge cases where the compiler can't infer intent, but the vast majority of manual memoization becomes unnecessary. Focus on writing clear, readable code and let the compiler handle performance.
