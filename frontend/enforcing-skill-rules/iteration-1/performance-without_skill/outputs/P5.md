# P5: Should I wrap everything in useMemo and useCallback right away?

Not everything needs memoization. useMemo and useCallback have their own overhead, so only use them when you have expensive computations or when passing callbacks to memoized child components.

Use React DevTools Profiler to identify actual performance bottlenecks before adding memoization. In many cases, the performance issue is elsewhere (unnecessary re-renders, large DOM, etc.).
