# G1: Should I add 'use client' to all my components to be safe?

**No, absolutely not.** `"use client"` is NOT needed for every component — only when using hooks, event handlers, or browser APIs. Over-marking components with `"use client"` kills SSR benefits, since it forces those components and their entire subtree to render on the client side.

Keep components as Server Components by default. Only add `"use client"` when the component genuinely needs interactivity (useState, useEffect, onClick handlers) or browser-only APIs (window, localStorage, IntersectionObserver).

This preserves streaming, reduces JavaScript bundle size, and keeps data fetching on the server where it belongs.
