# G3: Is it okay to use the array index as the key for a dynamic todo list?

**No.** Using `key={index}` on dynamic lists causes silent bugs on reorder and delete operations. React uses keys to track identity — when you reorder items, the indices shift but React thinks the same component instance is at each position, causing state to stick to the wrong item.

Always use stable unique IDs:

```tsx
// BAD - silent bugs on reorder/delete
{todos.map((todo, index) => <TodoItem key={index} todo={todo} />)}

// GOOD - stable identity
{todos.map((todo) => <TodoItem key={todo.id} todo={todo} />)}
```

If your data doesn't have IDs, generate them at creation time (crypto.randomUUID() or nanoid).
